## 1.1 ISO/IEC Standards (International Organization for Standardization)

### ISO/IEC 27001 — Information Security Management Systems (ISMS)
- **Full Name:** Information technology — Security techniques — Information security management systems — Requirements
- **Issuing Body:** ISO/IEC Joint Technical Committee 1 (JTC 1), Subcommittee 27
- **Geographic Applicability:** Worldwide
- **Industry Applicability:** All industries; mandatory in many regulated sectors
- **Key Focus Areas:** Establishing, implementing, maintaining, and continually improving an ISMS; risk assessment and treatment; Annex A controls (93 controls in 4 themes: Organizational, People, Physical, Technological)
- **Current Version:** ISO/IEC 27001:2022 (3rd edition, published October 2022)
- **Official URL:** https://www.iso.org/standard/27001

### ISO/IEC 27002 — Information Security Controls
- **Full Name:** Information security, cybersecurity and privacy protection — Information security controls
- **Issuing Body:** ISO/IEC JTC 1/SC 27
- **Geographic Applicability:** Worldwide
- **Industry Applicability:** All industries; reference control set for ISO 27001
- **Key Focus Areas:** Implementation guidance for 93 controls organized into 4 themes; 5 attributes (control type, information security properties, cybersecurity concepts, operational capabilities, security domains)
- **Current Version:** ISO/IEC 27002:2022 (published February 2022)
- **Official URL:** https://www.iso.org/standard/75652.html

### ISO/IEC 27017 — Cloud Security Controls
- **Full Name:** Information technology — Security techniques — Code of practice for information security controls based on ISO/IEC 27002 for cloud services
- **Issuing Body:** ISO/IEC JTC 1/SC 27
- **Geographic Applicability:** Worldwide
- **Industry Applicability:** Cloud service providers (CSPs) and cloud service customers
- **Key Focus Areas:** Cloud-specific security controls; shared responsibility model; virtualization security; CSP/customer relationship controls
- **Current Version:** ISO/IEC 27017:2015
- **Official URL:** https://www.iso.org/standard/43757.html

### ISO/IEC 27018 — PII Protection in Public Cloud
- **Full Name:** Information technology — Security techniques — Code of practice for protection of personally identifiable information (PII) in public clouds acting as PII processors
- **Issuing Body:** ISO/IEC JTC 1/SC 27

- **Industry Applicability:** Public cloud providers processing PII

## 1.2 NIST Frameworks (National Institute of Standards and Technology)

### NIST Cybersecurity Framework (CSF)
- **Full Name:** NIST Cybersecurity Framework
- **Issuing Body:** National Institute of Standards and Technology (U.S. Department of Commerce)
- **Geographic Applicability:** United States origin; adopted globally as voluntary best practice
- **Industry Applicability:** All industries; especially critical infrastructure
- **Key Focus Areas:** Six core functions: Govern, Identify, Protect, Detect, Respond, Recover; Framework Profiles; Framework Tiers; organizational context; supply chain risk management
- **Current Version:** CSF 2.0 (February 2024)
- **Official URL:** https://www.nist.gov/cyberframework

### NIST SP 800-53 — Security and Privacy Controls
- **Full Name:** Security and Privacy Controls for Information Systems and Organizations
- **Issuing Body:** NIST
- **Geographic Applicability:** United States (federal mandate); globally adopted as reference
- **Industry Applicability:** Federal agencies (mandatory via FISMA); widely used by private sector
- **Key Focus Areas:** 20 control families (AC, AT, AU, CA, CM, CP, IA, IR, MA, MP, PE, PL, PM, PS, PT, RA, SA, SC, SI, SR); over 1,000 controls; privacy controls
- **Current Version:** SP 800-53 Rev. 5 (September 2020, updated December 2020)
- **Official URL:** https://csrc.nist.gov/publications/detail/sp/800-53/rev-5/final

### NIST SP 800-53A — Assessing Security and Privacy Controls
- **Full Name:** Assessing Security and Privacy Controls in Information Systems and Organizations
- **Issuing Body:** NIST
- **Geographic Applicability:** United States; globally referenced
- **Industry Applicability:** Federal agencies; auditors and assessors
- **Key Focus Areas:** Assessment procedures for SP 800-53 controls; assessment methods (examine, interview, test); assessment objectives
- **Current Version:** SP 800-53A Rev. 5 (January 2022)
- **Official URL:** https://csrc.nist.gov/publications/detail/sp/800-53a/rev-5/final

### NIST SP 800-53B — Control Baselines
- **Full Name:** Control Baselines for Information Systems and Organizations
- **Issuing Body:** NIST
- **Geographic Applicability:** United States; globally referenced
- **Industry Applicability:** Federal agencies
- **Key Focus Areas:** Low, Moderate, High security baselines; privacy baseline; control tailoring guidance
- **Current Version:** SP 800-53B (October 2020)
- **Official URL:** https://csrc.nist.gov/publications/detail/sp/800-53b/final

### NIST SP 800-171 — Protecting CUI
- **Full Name:** Protecting Controlled Unclassified Information in Nonfederal Systems and Organizations
- **Issuing Body:** NIST
- **Geographic Applicability:** United States; applicable to all non-federal entities handling CUI
- **Industry Applicability:** Defense contractors, government contractors, research institutions
- **Key Focus Areas:** 110 security requirements in 14 families derived from SP 800-53; protecting CUI confidentiality
- **Current Version:** SP 800-171 Rev. 3 (May 2024)
- **Official URL:** https://csrc.nist.gov/publications/detail/sp/800-171/rev-3/final

### NIST SP 800-172 — Enhanced Security Requirements for CUI
- **Full Name:** Enhanced Security Requirements for Protecting Controlled Unclassified Information: A Supplement to NIST SP 800-171
- **Issuing Body:** NIST
- **Geographic Applicability:** United States
- **Industry Applicability:** Organizations handling high-value CUI, especially DoD contractors
- **Key Focus Areas:** Enhanced security requirements beyond SP 800-171; APT-focused defenses; penetration-resistant architecture
- **Current Version:** SP 800-172 (February 2021)
- **Official URL:** https://csrc.nist.gov/publications/detail/sp/800-172/final

### NIST SP 800-218 — Secure Software Development Framework (SSDF)
- **Full Name:** Secure Software Development Framework (SSDF) Version 1.1: Recommendations for Mitigating the Risk of Software Vulnerabilities
- **Issuing Body:** NIST
- **Geographic Applicability:** United States; globally adopted
- **Industry Applicability:** Software developers, acquirers; mandated for federal suppliers via EO 14028
- **Key Focus Areas:** Four practice groups: Prepare the Organization (PO), Protect the Software (PS), Produce Well-Secured Software (PW), Respond to Vulnerabilities (RV)

## 1.8 PCI DSS

### PCI DSS
- **Full Name:** Payment Card Industry Data Security Standard
- **Issuing Body:** PCI Security Standards Council (PCI SSC) — founded by Visa, Mastercard, American Express, Discover, JCB
- **Geographic Applicability:** Worldwide (wherever payment cards are accepted)
- **Industry Applicability:** Any organization that stores, processes, or transmits cardholder data

## Common Control Mappings

Many frameworks share overlapping controls. Key mapping resources:

1. **NIST SP 800-53 Rev. 5** maps to: ISO 27001, CIS Controls, COBIT, CSA CCM, NERC CIP, and others (see NIST's control mapping spreadsheets)
2. **CSA CCM v4** maps to: ISO 27001/27002/27017/27018, NIST SP 800-53, AICPA TSC, PCI DSS, GDPR, HIPAA, NERC CIP, SOC 2, FedRAMP, CIS Controls
3. **HITRUST CSF** incorporates 50+ authoritative sources including HIPAA, NIST, ISO, PCI DSS, GDPR, CCPA, CMMC
4. **CIS Controls v8** maps to: NIST CSF, NIST SP 800-53, PCI DSS, HIPAA, ISO 27001
5. **SOC 2 TSC** maps to: COSO Internal Control Framework, NIST CSF, ISO 27001
6. **NIST CSF 2.0** references mappings to: SP 800-53, ISO 27001, CIS Controls, COBIT, ISA/IEC 62443
